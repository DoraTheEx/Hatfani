


חלוקת כתובות IP: סטטית לעומת דינמית
כתובת IP היא מספר ייחודי המזהה מכשיר ברשת. חלוקת כתובות IP יכולה להתבצע בשתי דרכים עיקריות: סטטית ודינמית.

חלוקת כתובות IP סטטית:

כתובת IP מוגדרת ידנית לכל מכשיר ברשת.
היתרון: כתובת IP קבועה מאפשרת גישה קלה ופשוטה למכשירים ברשת, שימושית עבור שרתים, מדפסות רשת, התקנים IoT ועוד.
החיסרון: ניהול ידני מורכב, פחות גמיש, לא מתאים לרשתות גדולות ודינמיות.
חלוקת כתובות IP דינמית:

כתובת IP מוקצית באופן אוטומטי על ידי שרת DHCP (Dynamic Host Configuration Protocol).
היתרון: ניהול קל ופשוט, גמישות רבה, מתאים לרשתות גדולות ודינמיות.
החיסרון: כתובת IP אינה קבועה, עלולה להשתנות עם הזמן, פחות נוחה עבור גישה ישירה למכשירים.
תהליך DHCP:

גילוי: מכשיר חדש ברשת שולח הודעת DHCP Discover לכל המחשבים ברשת.
הצעה: שרת DHCP בודק אם כתובת IP זמינה ומציע אותה למכשיר.
בקשה: המכשיר שולח הודעת DHCP Request חזרה לשרת DHCP כדי לאשר את כתובת IP.
אישור: שרת DHCP מאשר את כתובת IP ומקצה אותה למכשיר לזמן מוגדר (חכירה).
חידוש: לפני תום זמן החכירה, המכשיר יכול לחדש את כתובת IP על ידי שליחת הודעת DHCP Request נוספת.
שחרור: כאשר המכשיר מתנתק מהרשת, הוא משחרר את כתובת IP חזרה לשרת DHCP.
